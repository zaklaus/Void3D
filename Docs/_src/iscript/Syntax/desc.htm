IScript language syntax and usage

This page deals about syntax of IScript scripting language. Entire language originates at C, so only changes against C will be discussed.

Enhanced function parameters

Functions are defined the same way as in C – list of parameters is in () braces. What changed is that:
1. default parameters may be supplied on function declaration

2. default parameters may be specified for any parameters, not only continuous last parameters (as in C++)

3. when calling function, two modes are possible:

A)      classic way – specifying parameters in order as they appear in declaration

B)      pass parameters by specifying parameter name, followed by ‘:’ and value, for example:

Function declared as:
void MyFunc(int speed = 20, float length, byte code = 9);

Function call with parameter naming:
MyFunc(length: 14.2, code: 7);

Note that order of parameters doesn’t matter in this mode, but all parameters which don’t have default values specified must be listed.
There’s no run-time penalty for choosing first or second parameter passing mode.

Tables

Tables are an enhancement in script language, allowing to specify table template used with ITabler interface. There may be always maximally one table specified in each script. The syntax of table template is as follows:

table <”quoted_table_name”> {
            <table_element> [ , <table_element> ]
}

table_element is one of following:

1. <variable_type>  <variable_name>  [ [ <array_size> ] ]  <”quoted_variable_name”> [ ( <default_value>  [ , <min_range>, <max_range>  ]   ) ] ;
2. string <variable_name>  [ [ <array_size> ] ]  <”quoted_variable_name”> ( <max_string_size> );

Additionally, several elements may be enclosed in logical branch, for easier editing in table editor, using following construct:
branch <”quoted_name”> {
            <table_element> [ , <table_element> ]
}
The branch block may be used anywhere inside of table construct, and may be nested as desired.

The items in [] braces are optional.
quoted_table_name … user-friendly name of table shown in table editor
variable_type … type (char, int, etc)
array_size … size of array
variable_name … unique name of variable accessed from within script code
quoted_variable_name  … user-friendly name of variable shown in table editor
default_value … number specifying default value assigned to table element
min_range, max_range … editable range, used in table editor
max_string_size … integer specifying maximal size a string may have, including terminating null character

The table construct is used for creation of table template in C_script interface. Additionally, if script contains table template, a virtual machine (C_v_machine) loading the script creates a table using the template, and initializes it into default values.
Application may access members of the table externally using virtual machine’s table. Script code may access all script variables as read-only variables. Writing into table using script code is not allowed.

No pointers

The language doesn’t support pointers.

Natural string type

To support strings, the language introduces string keyword. Strings may be declared either as table elements, global variables, or string literals (passed into functions as parameters). Strings are always read-only, and are only usable as parameters into function calls.
For function externally linked to script code, the string variable appears as pointer to null-terminated char array.

Limited arrays

Arrays are not supported on variables, except of table elements. If script code attempts to read out-of-array variable from table, the running of script code is stopped, and error code is returned to calling application.

No cast operator

Casting operator is unsupported.

Number syntax

Octal numbers are not supported.

Typedef changes

typedef keyword is more strict than in C. Assignment into type declared using typedef is allowed only from exactly the same type. Anyway, assignment from typedef’d type into other compatible type is allowed.

Removed keywords

Following keywords are removed from the language:

static, auto, register, sizeof, volatile, signed, unsigned, double, union, goto

 

Linking script program with external functions

For linking script code with external functions, which are called from within the script, before using such function, it must be declared (unlike in C, where parameters are deduced if declaration was not seen). Declare function the same way like in C, using function return type, name and parameter list, terminated by semicolon.

The difference against C is that once a function has been declared without body, it is assumed to be extern, so body cannot be provided later in script. When function declared as external is used in script, the C_v_machine interface tries to link with it during loading the script, so its address and name must be provided in array if link symbols. Once script is loaded in virtual machine, the call of external function is performed normally.

Note that there’s no mechanism for assuring that actual number and types of parameters passed to external function match the declaration (like name mangling in C++), so it’s programmer’s responsibility to assure that function declaration seen by script code matches actual function defined in application.

The script library uses stdcall calling convention when calling external function, so external function must be defined with __stdcall keyword in order to match the order of parameters and way how stack is cleaned after function call.
