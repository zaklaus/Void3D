C_v_machine::Load

Load program from script.

ISL_RESULT Load(

            CPC_script script,

            const VM_LOAD_SYMBOL *symbols

);

Parameters:

script

Pointer to C_script interface containing the compiled script to load.

symbols

Pointer to an array of external symbols used for linking with the loaded script. The pointer may be NULL.
Each element of array is a VM_LOAD_SYMBOL structure, which defines name of symbol and its address. The array must be terminated with element which has address set to NULL.
The symbol table may be made of chains of arrays - if name is NULL and address is not NULL, the address member of element is interpreted as a pointer to another array of VM_LOAD_SYMBOL structures.

Return value:

The returned value is I3D_OK if the call was successful.

See also:

Unload, VM_LOAD_SYMBOL

C_v_machine::Run

Run loaded program.

ISL_RESULT Run(

            dword *ret_val,

            void *start,

            dword flags,

            dword num_pars=0,

             ...

);

Parameters:

ret_val

Pointer to dword value, which will contain the returned value from the called script function. This parameter may be NULL.
If the called function doesn’t return value, the value written to pointed variable is undefined.

start

The beginning address, where execution starts. The type of this variable is dependent on flags passed to the function. See flags parameter for details.

flags

Flags specifying behavior. Following flags are defined:

VMR_FNCNAME

The start parameter is a string specifying name of script function which should be called.

VMR_ADDRESS

The start parameter is a pointer to script function. The address must be determined by GetAddress method.

VMR_SAVEDCONTEXT

The start parameter is a dword value specifying ID of saved context, where execution should continue. When script is run this way, the num_args parameter must be zero.

num_pars

Number of parameters passed to the function, which will be passed to the called script function. The parameters are passed right after this parameter.

…

Variable number of parameters passed to the called function.

Return value:

The returned value is I3D_OK if the call was successful, or other error code specifying type of error which occurred.
Additionally, the returned value may be a success code of following values:
ISL_SUSPENDED … marking the execution of script has been suspended using the SaveContext method called during the execution of the Run method.

See also:

SaveContext

C_v_machine::GetAddress

Return address of script’s symbol in global scope. The symbol may be either variable name, or function name.

void *GetAddress(

            const char *sym_name

) const;

Parameters:

sym_name

Name of symbol, whose value is queried.

Return value:

The returned value is const pointer to memory, where function/variable begins. If the symbol was not found, the returned value is NULL.
If the symbol is name of variable, it may be written to. If writing occurs into code segment (the symbol name is name of function), the script may become corrupted.

C_v_machine::EnumSymbols

Enumerate all symbols defined in loaded script.

ISL_RESULT EnumSymbols(

            void (ISLAPI *cb_proc)(const char *name, void *address, bool is_data, void *context),

            void *cb_context

) const;

Parameters:

cb_proc

Callback function which will be called with each enumerated symbol. This function is defined as:

void ISLAPI EnumSymbols(
  const char *name,
  void *address,
  bool is_data,
  void *context
);

The callback function’s parameters are as follows:

name … symbol name
address … symbol address
is_data … boolean vaue set to true if symbol is data, and to false if symbol is function
context … context value passed to the function call

cb_context

User-defined context value, which will be passed to the callback function.

Return value:

The returned value is I3D_OK if the call was successful, or ISLERR_NOPRG if no program is loaded.

C_v_machine::SaveContext

Save current context - stack variables of running program. This method may be called only from running program - from an extern function called by script.
The method returns ID value, which identifies saved context block (usable only with this interface). This ID value may be used as a parameter to Run() function, with VMR_SAVEDCONTEXT flag, for continuing in the running of program. Note that after continuing execution of suspended script using Run method, the ID becomes invalid and may be re-used by SaveContext again.

The ID value 0 is an invalid context value, and will never be returned by IScript library, so it may be used by application to mark NULL (non-assigned) context ID.

ISL_RESULT SaveContext(

            dword *ret_id

);

Parameters:

ret_id

Pointer to variable, which will contain valid context ID if the call succeeds.

Return value:

The returned value is I3D_OK if the call was successful, or ISLERR_GENERIC if the context was already saved. See Remarks for details.

Remarks:

This method may be called only from functions called by script code, which are executed using Run method of this interface. The method is used for stopping processing of script code (freezing it), and return from Run method immediately with ISL_SUSPENDED return code.

See also:

Run, ClearSavedContext

C_v_machine::ClearSavedContext

Clear saved context, identified by ID. After calling this method, the ID value passed in becomes invalid, and may be reused in subsequent calls in SaveContext method.

ISL_RESULT ClearSavedContext(

            dword id

);

Parameters:

id

ID identifying saved context. This value is generated by SaveContext method.
Additionally, this value may be set to zero to indicate that all saved contexts of this script should be cleared.

Return value:

The returned value is I3D_OK if the call was successful, or ISLERR_NOT_FOUND if saved context with specified ID was not found.

See also:

SaveContext

C_v_machine::GetSavedContext

Get saved context data into a buffer. This method is suitable for permanently save current saved context (for example onto disk), and later restored into exactly the same state. This functionality may be utilized for SaveGame feature of game.

ISL_RESULT GetSavedContext(

            dword id,

            C_buffer<byte> &buf

) const;

Parameters:

id

ID identifying saved context. This value is generated by SaveContext method.

buf

Buffer where all relevant data of saved context will be stored. The format of the buffer is implementation-specific, and is meant only for use in SetSavedContext call.

Return value:

The returned value is I3D_OK if the call was successful, or ISLERR_NOT_FOUND if saved context with specified ID was not found.

See also:

SetSavedContext

C_v_machine::SetSavedContext

Create saved context data from buffer created by GetSavedContext.

ISL_RESULT SetSavedContext(

            dword *ret_id,

            const byte *buf,

            dword buf_size

);

Parameters:

ret_id

Pointer to variable, which will contain valid context ID if the call succeeds.

buf, buf_size

Pointer to buffer containing data created by GetSavedContext call on a virtual machine with the same script loaded.

Return value:

The returned value is I3D_OK if the call was successful, or ISLERR_INVPARAM if the content of the buffer is corrupted.

See also:

GetSavedContext
