Reference counting

Most interfaces in Instanity3D library use reference counting method for keeping life-time of the interface. When an interface is created, its reference counter is set to one.
Each time application calls AddRef method of reference-counted interface, the counter is increased by one.
When application call Release on interface, the counter is decreased by one. When counter reached zero in call to Release, the interface is destroyed.

Note that if your program calls Release on interface, which it hasn’t created, or on which it hasn’t called AddRef, the interface may be unexpectedly destroyed while some other parts of code are still using it.
Similarly, if program calls AddRef without matching Release during its run-time, the interface may never be destroyed, resulting in memory leaks.

Common methods of reference-counted interfaces:

unsigned long AddRef();

This method increases reference counter of interface by one.

Return value:

The returned value is new reference counter.

unsigned long Release();

This method decreases reference counter of interface by one. When the counter reaches zero during this call, the interface is destroyed.

Return value:

The returned value is new reference counter.

C_unknown

C_unknown is a base class with implemented AddRef and Release methods, and a protected virtual destructor. This class may be used by application classes to implement reference-counting classes.

Header file: C_unknown.hpp

 

Remarks:

The reference counting is also designed for use by smart pointers.
