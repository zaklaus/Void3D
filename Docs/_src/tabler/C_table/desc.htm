C_table::Load

Load data. The source of loading is dependent on flags passed to the function.

bool Open(

            const void *src,

            dword flags

);

Parameters:

src

Source for loading. This pointer is interpreted depending on flags parameter.

flags

Loading flags. One of following flag may be specified:

TABOPEN_TEMPLATE

Load table from template, and initialize all data to default values. Parameter src is a pointer to C_table_template class.

TABOPEN_FILENAME

Load table from file. Parameter src is a pointer string containing name of the file.

TABOPEN_FILEHANDLE

Load table from opened file. Parameter src is a pointer to C_cache interface.

TABOPEN_DUPLICATE

Duplicate another table. Parameter src is a pointer to C_table interface.

Additional flags (usable in combination with above flags):

TABOPEN_UPDATE

This flag indicates, that if the table is already loaded, then the layout of the data is not changed, but rather only matching data from the source are updated in the table. This is used for example when table template has changed, and application needs compatibly load and fill all non-changed data.
If this flag is used when table is currently not loaded, it is ignored.

Return value:

The returned value is true if table was loaded successfully.

See also:

Save, C_table_template, C_cache

C_table::Save

Save data. The destination for saving is dependent on flags passed to the function.

bool Save(

            const void *dst,

            dword flags

) const;

Parameters:

dst

Destination for saving. This pointer is interpreted depending on flags parameter.

flags

Saving flags. One of following flag may be specified:

TABOPEN_FILENAME

Save table to file. Parameter dst is a pointer string containing name of the file.

TABOPEN_FILEHANDLE

Save table to opened file. Parameter dst is a pointer to C_cache interface.

Return value:

The returned value is true if table was saved successfully. If table is not loaded, or specified destination cannot be created/opened, the returned value is false.

See also:

Load, C_cache

C_table::IsLoaded

Check if data are currently loaded in the table.

bool IsLoaded() const;

Return value:

The returned value is true if table contains loaded data.

See also:

Load

C_table::IsModified

Check if any data was modified (using the build-in editor) since loading the table.
Note that this method doesn’t detect if user changes data contents by directly writing to the table. It detects only changes made in built-in editor.

bool IsModified() const;

Return value:

The returned value is true if data was modified.

C_table::NumItems

Get number of data items contained in table.

dword NumItems() const;

Return value:

Number of items accessible in table. This is the maximal data index plus one, which may be passed into data access methods.
When table is not loaded, this method returns zero.

C_table::ArrayLen

Get size of array of particular data element.

dword ArrayLen(

            dword index

) const;

Parameters:

index

Data element index.

Return value:

The returned value is the length of array of given data element. If table is not loaded, or index is out of range, the returned value is 0xffffffff. If given data element is not an array, the returned value is 1.

C_table::StringSize

Get size of string of particular data element.

dword StringSize(

            dword index

) const;

Parameters:

index

Data element index.

Return value:

The returned value is maximal length of string of given data element (including terminating null character). If table is not loaded, or index is out of range, or given element is not a string type, the returned value is 0xffffffff.

C_table::SizeOf

Get size of particular data element type, in bytes.

dword SizeOf(

            dword index

) const;

Parameters:

index

Data element index.

Return value:

The returned value is the length of data type of particular data element. If the type is string, the returned value is equal to value returned by StringSize method. If table is not loaded, or index is out of range, the returned value is 0xffffffff.

C_table::GetItemType

Get size of particular data element type, in bytes.

E_TABLE_ELEMENT_TYPE GetItemType(

            dword index

) const;

Parameters:

index

Data element index.

Return value:

The returned value is the type of data type of particular data element. If table is not loaded, or index is out of range, the returned value is TE_NULL.

C_table::Edit

Edit data in the table, using built-in editor.

void *Edit(

            const C_table_template *templ,

            void *hwnd_parent,

            table_callback cb_proc,

            dword cb_context,

            dword create_flags = 0,

            int *pos_size = NULL

);

Parameters:

templ

Pointer to table template, which specifies data layout. This must be the same template, which was used for Load method, because the template is not internally held in the table due to memory saving. See C_table_template for description of table template.

hwnd_parent

Handle of window (HWND), which will be used as parent window for the editor window. This parameter may be NULL.

cb_proc

Callback function, which will be used during editing. See description of table_callback for details. This parameter may be NULL.

cb_context

Context value which will be passed to the callback function.

create_flags

Creation flags. Following flags are currently defined:

TABEDIT_CENTER

Center the editor dialog window, relative to screen.

TABEDIT_MODELESS

If this flag is specified, a modeless dialog box is created, and function returns its handle window (HWND).
If not specified, a modal dialog is created, and program will continue when user closes the edit dialog.

TABEDIT_IMPORT

Enable the Import button at bottom of the table.

TABEDIT_EXPORT

Enable the Export button at bottom of the table.

TABEDIT_HIDDEN

Create the window hidden. This flag has meaning only for modeless dialog mode, because this way application may position or size the window before showing it using the returned window handle.

TABEDIT_CONTROL

Create window, which is suitable as a child of another window. When this flag is specified, the window will not have system bar and status bar, and window border. This flag is ignored for modal dialog mode.

pos_size

Pointer to four integer values, specifying the creation position and size of dialog box. The order of elements in the array is following: position_x, position_y, size_x, size_y.

Return value:

If modal dialog box was requested, the returned value is always NULL.
If modeless dialog box was requested, the returned value is handle of editor window (HWND), or NULL if some error occurred.

See also:

C_table_template, table_callback

C_table::*Item*

Get access to data element stored in the table.

Access to raw data:

void *Item(dword index);

 

Access to data by reference:

bool &ItemB(dword index);
int &ItemI(dword index);

float &ItemF(dword index);

byte &ItemE(dword index);

char *ItemS(dword index);

S_vector &ItemV(dword index);

 

Retrieving data using const methods:

bool GetItemB)(dword index) const;

int GetItemI(dword index) const;

float GetItemF(dword index) const;

byte GetItemE(dword index) const;

const char *GetItemS(dword index) const;

const S_vector &GetItemV(dword index) const;

 

Access to arrays by reference:

bool &ItemB(dword index, dword ai);
int &ItemI(dword index, dword ai);

float &ItemF(dword index, dword ai);

byte &ItemE(dword index, dword ai);

char *ItemS(dword index, dword ai);

S_vector &ItemV(dword index, dword ai);

 

Retrieving array members using const methods:

bool ItemB(dword index, dword ai) const;
int ItemI(dword index, dword ai) const;

float ItemF(dword index, dword ai) const;

byte ItemE(dword index, dword ai) const;

const char *ItemS(dword index, dword ai) const;

const S_vector &ItemV(dword index, dword ai) const;

Parameters:

index

Data element index.

ai

Array index – index of element within its array.

Return value:

The returned value is:

- raw pointer to memory for Item method
- reference (writtable) to element for Item* methods
- element value for GetItem* methods

Remarks:

These functions are designed for accessing one element at a time. When using functions returning reference to the element, this reference may be used to write to the element (but also for reading the element). For more optimal retrieving of element, use GetItem* methods.
