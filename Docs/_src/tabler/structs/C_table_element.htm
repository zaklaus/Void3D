C_table_element

C_table_element class is used for definition of single data element of table. Arrays of these classes are used for definition of table layout. This class is designed to work with C_table_template class.

class C_table_element{
public:
   E_TABLE_ELEMENT_TYPE type;
   dword tab_index;
   const char *caption;
   union{
      int flt_min;
      int int_min;
      dword branch_depth;
      char *enum_list;
      dword array_size;
      dword string_size;
   };
   union{
      int flt_max;
      int int_max;
      const char *branch_text;
   };
   union{
      float flt_default;
      int int_default;
      char *string_default;
   };
   const char *help_string;
};

typedef C_table_element *PC_table_element;
typedef const C_table_element *CPC_table_element;

Members:

type

Type of data element. The type TE_NULL has a special meaning, and is used in the last element of an array of elements to mark the end of the array.

In case of TE_ARRAY, the tab_index member used with the TE_ARRAY size must equal to first data element. Multi-dimensional arrays are allowed, but there’re no direct functions for accessing the elements of such arrays, so user is responsible for computing element offsets directly, using raw memory pointer into first element.

Arrays of TE_BRANCH items are supported, (similar to concept of arrays of structures in C), where each element enclosed in the branch has the same array size as specified for the TE_ARRAY element. Anyway, the elements inside of the branch are unrolled – they all appear as separate arrays of the same type, not as an array of structures.

tab_index

Index, by which data of this element will be accessed. For TE_BRANCH type this member is ignored. For TE_ARRAY type, the index must equal to index of next element (array member type).

caption

User-friendly string, describing the name of element. This string is displayed in table editor with the element’s field.

flt_min, int_min, flt_max, int_max

Minimal and maximal values of a range for integer or float numbers. The range is used for displaying slider in the editor. Applicable to TE_INT and TE_FLOAT types.

branch_depth

Used with TE_BRANCH type. It specifies how many elements are inside of the branch.
If TE_BRANCH or TE_ARRAY elements are nested in another branch, they are counted as one element.

enum_list

Strings specifying names of members of enumerated type. The names are stored in one null-terminated string, separated by null character, and entire string is terminated by another null character. Applicable to TE_ENUM types.
If this member is NULL, the editor posts TCM_GET_ENUM_STRING to the application callback message.
Note that this member is aliased in union with other members, starting with int, so you must explicitly cast the string to int, for example: (int)”One\0Two\0Three\0”.

array_size

Size of array of elements. Applicable to TE_ARRAY type.

string_size

Maximal size of string, including terminating null character. Applicable to TE_STRING type. This member must be greater than zero.

branch_text

Additional description text in branch, displayed in the text editor next to element caption. Applicable to TE_BRANCH type. Special character % followed by number in [] braces in this string causes special formatting when displaying the text. Each occurrence of %[n] will expand into current value of sub-element of the branch, indexed from zero. For example you may use this text for displaying second sub-element in the branch: “Name: %[1]”.
Note that this member is aliased in union with other members, starting with int, so you must explicitly cast the string to int, for example: (int)”Factory %[0]”.

flt_default, int_default, string_default

Default value assigned into table element, when table is loaded using the template. Applicable to TE_INT, TE_FLOAT, TE_BOOL, TE_STRING and TE_ENUM types. The default values are useful for initialization of empty tables to some reasonable default values.
Note that these members are all aliased in union together, starting with float, so simply pass in desired value, letting compiler automatically perform conversion to float (except of string, which must be explicitly converted to float bitmask).

help_string

Help string associated with the element. This may be long description text of the element (since space for caption is very limited). When user presses F1 during editing, the text of focused element is displayed in separate pop-up window.
