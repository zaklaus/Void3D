Tabler library

Tabler library is a database system, which is capable of holding certain amount of data and easy access that data using user-friendly programming interface. The data may be saved into a file, and loaded back. Moreover, for insertion and editing of the data, library provides dialog box, which allows comfortable editing of the data.

The core of the library is C_table inteface, which serves as storage holder for data, and also allows editing of the data it contains.
The flexibility of data stored in the interface is assured by using customizable templates (technically arrays of structures written in C/C++), which allow definition of any data layout that are required.

Data layout

The data in the table are identified by indices. Each data element is identified by certain index, starting from zero up to maximal data index. The indices are defined in table template, which used for initialization of the table. Once data layout is set (using Load method), it cannot be changed other way than re-loading the table using different template.

The interface contains number of data access functions, each accepting index of data, which is being read or written. The table template also specifies the type of data (one of above), which is stored under particular index. The indices don’t need to be used contiguously, and there may be holes in their usage. Anyway it is advised to keep the maximal used index small, in order to save memory used by the interface.

 

Editing

The contents of loaded table may be comfortably edited using the built-in editor, which displays a dialog box with list of elements. Some element blocks may be collapsed in expandable tab – this behavior is defined by using branches.

The system supports concept of invalid values for several data types, which means that some special value is assigned the role to represent an invalid (or unassigned) value. The meaning of such values is that they tell the editor to display undeterminable mark during editing, informing the user that value is special. As soon as user types in some reasonable value, the value will be displayed normally.

The special value numbers are defined as follows:

#define TAB_INV_INT 0x80000000
#define TAB_INV_FLOAT 1e+38f
#define TAB_INV_BOOL 2
#define TAB_INV_STRING "\1"
#define TAB_INV_ENUM 0xff

You may write these values into elements using the data access methods of the C_table interface.
