IPH_world

This interface is the master interface managing physics. It is used for setting global parameters, creating other interfaces, and updating the physical world.

IPH_world::CreateJoint

 

 

Create new joint.

 

PIPH_joint CreateJoint(

            IPH_JOINT_TYPE type

);

 

Parameters:

 

type

 

Requested joint type. Following types are allowed:

 

IPHJOINTTYPE_BALL

 

This is a ball and socket joint type, allowing rotational movement between two bodies, but keeping fixed position.

 

IPHJOINTTYPE_HINGE

 

Hinge joint, limiting positional movement, and fixing one rotational axis.

 

IPHJOINTTYPE_SLIDER

 

Slider joint, limiting rotational movement, and allowing positional movement along one axis.

 

IPHJOINTTYPE_UNIVERSAL

 

Universal joint, limiting positional movement, and allowing rotation along two independent axes.

 

IPHJOINTTYPE_HINGE2

 

Hinge2 is a joint limiting positional movement, and allowing rotation along two axes. This joint is suitable for steering wheels bodies.

 

IPHJOINTTYPE_FIXED

 

Fixed joint. Used for debugging purposes only. Limiting any movement between two bodies.

 

Return value:

 

The returned value is pointer to newly created joint class.

 

See also:

 

IPH_joint

            CPI3D_volume vol

) const;

 

Parameters:

 

vol

 

Pointer to I3D_volume interface, which is assumed to be part of physics body object.

 

Return value:

 

The returned value is pointer to IPH_body interface, to which given volume belongs, or NULL of volume doesnt belong to any body.

Determine if specific joint type is allowed to be attached only to single body. Usually joints must be attached to two bodies for proper work.

            IPH_JOINT_TYPE type

) const;

 

Parameters:

 

type

 

Joint type, which is being queried.

 

Return value:

 

The returned value is true if given joint type is allowed to be attached to single body only.
