ISound flags

These flags are common for all ISound interfaces and methods which accept dword flags parameter. You may use combination of following flags.

Flag bits:

ISND_STREAMING

The sample will be played in streaming mode. In this case, the wave file is only opened, and open file handle is saved. The file will be read as sound is being played.
This flag is applicable in call to ISND_source::Open method.

ISND_NOPANNING

Panning of sample will not be adjustable (always playing at the center). This is a performance flag, which you may use, if you are sure, you will not change volume of the sample.

ISND_PRELOAD

This flag specifies that sound data should be pre-loaded into system memory, rather than DirectSound sound buffer. When this method is used, sound data are stored in sound source buffers in system memory, decompressed and in target sound format. This saves sound hardware memory, and opening sound from such pre-loaded source is optimized to be fast.
This flag is applicable in call to ISND_source::Open method.

ISND_FREQUENCY

Adjustable frequency requested. Specify this, if you want adjust playback frequency of the sample. Note that if playback frequency of sound is different than playback frequency of primary buffer, depending on sounds card, mixing may become spending more CPU time.
By default, sounds have un-adjustable frequencies, and they play on frequency specified in source data. By specifying this flag you may change playback frequency using the ISND_sound::SetFrequency method.

ISND_CARDMEM

Locate sound buffer in hardware memory. If hardware resource allocation fails, the call with this flag also fails.

ISND_SYSMEM

Locate sound buffer in system memory. If system memory allocation fails, the call with this flag also fails.

ISND_LOOP

Play sound looped.
This flag is applicable in call to ISND_sound::Play method.

ISND_3DBUFFER

Use 3D hardware resources. If hardware resource allocation fails, the call with this flag also fails.

Remarks
Not all flags are applicable by all methods. If flags not supported by method are passed to its call, the system returns ISNDERR_INVALIDPARAM.
