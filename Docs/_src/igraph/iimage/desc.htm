IImage::Open

This method initializes the content of IImage interface. This means allocation of surface in requested pixel format and dimensions, and filling it with data.

bool Open(

            const char *filename,

            dword flags = 0,

            dword sx = 0,

            dword sy = 0,

            const S_pixelformat *pf = NULL,

            dword num_mip_levels = 0

);

 

Parameteres:

 

filename

 

Pointer to null-terminated filename, which is used to initialize the interface. It may be NULL if empty image is requested.
The filename must specify valid image in one of formats supported by IGraph library.

 

flags

The initialization flags. This may be combination of following flags:

IMGOPEN_TEXTURE

The created surface is intended for use as a D3D texture.

IMGOPEN_NOREMAP

When specified, no conversion of source pixel format is performed. The format of input data is used.

IMGOPEN_DITHER

Dithering may be applied during image conversion to destination pixel format, for enhancing of quality.

IMGOPEN_SYSMEM

The surface is requested in system memory. If this kind of memory is unavailable, the creation may fail.

IMGOPEN_VIDMEM

The surface is requested in video memory. If this kind of memory is unavailable, the creation may fail.

IMGOPEN_EMPTY

Empty surface will be created.

IMGOPEN_MIPMAP

Request for creation of mipmap chain. This flag is used with IMGOPEN_TEXTURE.

IMGOPEN_COLORKEY

Implicitly encodes color-key information into surface. This flags is used with compressed pixel formats supporting color-keying in pixels.

IMGOPEN_HINTDYNAMIC

Specifies that the texture will be often locked and modified.

IMGOPEN_HINTSTATIC

Specifies that the texture will be seldom locked (ideally once during filling of data).

IMGOPEN_MEM_PTR

Map specified by memory pointer, rather than by filename - used to read bitmaps from resources - currently Windows BMP format is supported.

IMGOPEN_RENDERTARGET

The created surface will be possible to use as a render target.

IMGOPEN_INIT_CKEY

This method initializes color-key information from pixel on position 0 (palletized source), or black color (RGB source). Also RLE transparency mask is created.

sx, sy

 

The size of surface. These parameters are used only if IMGOPEN_EMPTY flag is specified. If this flag is not specified, dimensions of surface are determined from format of opened bitmap.

 

pf

 

Pointer to requested pixel format of surface. This parameter is used if IMGOPEN_TEXTURE flag is specified. See S_pixelformat for details.

 

num_mip_levels

 

The number of mipmap levels being created, not including the first level. This parameter is used if IMGOPEN_MIPMAP flag is specified.

           

Return value:

 

If the call succeeds, the returned value is true.
If the call fails due to low-memory conditions, file open fail, or other reason, the returned value is false.

IImage::Lock

 

This method is used for obtaining access to area of IImage interface. The surfaces remains locked until calling the Unlock method.

 

bool Lock(

            void **area,

            dword *pitch,

            bool read_only = true

);

 

Parameteres:

 

area

 

The pointed variable will contain pointer to surface data. The memory may be read from or written to.

 

pitch

The pointed variable will contain pitch of single scan-line of buffer, in bytes.

read_only

When true, read-only access to surface is requested, otherwise read/write access is requested.

Return value:

 

The method returns true if the call succeeds, or false, if the surface cannot be locked for some reason.

 

Remarks:

The image memory is allocated as rectangular block, the width of single scan-line must be determined by pitch parameter.
If surface is in compressed pixel format, the function returns the pitch of one line of 4x4-pixel compressed blocks.

 

See Also:

 

Unlock

IImage::AlphaBlend

This method is used for conversion of RGB image data into alpha channel of the surface data. The conversion is made from specified source IImage interface into this interface.

bool AlphaBlend(

            const IImage *lp_IImage,

            dword flags = 0

);

 

Parameters:

lp_IImage

Pointer to IImage interface which holds RGB data used as source of conversion.

flags

This parameter may be combination of following flags:

IMGBLEND_DITHER

Perform dithering during conversion. This improves quality of alpha channel during conversion to lower bit-depth.

IMGBLEND_INVALPHA

The alpha value is inverted prior to writing it into the surface.

Return value:

If the call succeeds, the returned value is true.

Remarks:

This IImage interface must be already initialized and its pixel format must contain alpha channel. The dimensions of source and destination images must match.
The conversion is performed using the weighted sum of RGB values of each pixel and storing it into the alpha channel of destination surface.

IImage::ApplyFunction

This method is used for user-defined conversion of image data. The method applies a function onto every pixel in the image, in one or more passes (this way, caller may use one pass for retrieving data, and other passes for modifying data.
All pixel-format conversions are handles inside of the function, and caller deals only with pixels in ARGB format.

bool ApplyFunction(

            t_ApplyFunc *func,

            dword num_passes = 1

);

 

Parameters:

func

Pointer to callback function, which will be called for each pixel in the image. The function prototype is declared as:

typedef dword I2DAPI t_ApplyFunc(
        dword argb,
        dword x,
        dword y,
        dword pass_index
);

argb  current pixel in ARGB format
x, y  coordinates of the current pixel
pass_index  index of current pass

The returned value must be new pixel value stored in the image.

num_passes

Number of passes. For each pass, the func will be called for all pixels in the image.

Return value:

If the call succeeds, the returned value is true.

Parameters:

IImage::GetPal

 

This method returns pointer to palette associated with surface of the IImage interface.

 

dword *GetPal() const;

 

Return value:

 

If pixel format of the surface is palletized, the returned value is pointer to array of 256 entries of color palette associated with the surface, in ARGB format.

If surface is not initialized, or pixel format is not palletized, the returned value is NULL.

 

See Also:

 

SetPal, GetPixelFormat

bool SetPal(

);

 

Parameters:

           

palette

 

Pointer to array of 256 dword values, where each value specifies ARGB palette entry.

 

Return value:

Otherwise the returned value is false.

IImage::GetColorkeyInfo

Using this method, applications can get access to RLE-compressed colorkey information stored with IImage interface, if it was opened using the IMGOPEN_INIT_CKEY flag.

dword GetColorKeyInfo(

            dword *size = NULL

);

Parameters:

size

Pointer where size of the colorkey information will be written to. You may pass NULL if you do not wish to receive this information.

Return value:

The returned value is a pointer to a RLE-compressed colorkey information, associated with pixels in images surface.

IImage::CopyStretched

This method is used for copying contents of source image surface into this images surface. The pixel formats of both images must match.

bool CopyStretched(

            const IImage *src_img

);

 

Parameters:

 

src_img

 

Pointer to IImage interface, whose data will be copied to this interface.

 

Return value:

 

If the call succeeds, the returned value is true.

 

Remarks:

 

Dimensions of both images doesnt need to match. The surface data from source image are copied into destination image stretched, which means that the image is rescaled to fill entire contents of destination surface.
