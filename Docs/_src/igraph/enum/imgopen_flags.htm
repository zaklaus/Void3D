Image opening flags

Image opening flags is set of flags which may be specified during opening an image. They specify how should be the image opened.
Following flags are defined:

IMGOPEN_TEXTURE

Create texture surface to hold image data. This flag is specified, if image is a placeholder for texture used for 3D rendering.

IMGOPEN_NOREMAP

When this flag is specified, the library finds appropriate format for holding image data, compatible with underlying hardware.
When specifying this flag, no conversion is performed, and the image is opened in the format in which it is stored in the file.

IMGOPEN_DITHER

Dither image pixels during conversion to target pixel format. Dithering is used to enhance image quality while converting to pixel formats with less bits of precision, for example during conversion from 240bit true-color format to 16-bit high-color format.

IMGOPEN_SYSMEM

Instructs the system to create surface in system memory. If creating surface in this kind of memory fails, the image opening call fails.

IMGOPEN_VIDMEM

Instructs the system to create surface in video memory. If creating surface in this kind of memory fails, the image opening call fails.

IMGOPEN_EMPTY

Create empty, non-initialized surface. The image is opened by specified dimensions, but no contents is loaded from file.

IMGOPEN_MIPMAP

Automatically generate mipmap levels when opening image. The system creates mipmap levels and initializes them with filtered versions of the image being opened.

IMGOPEN_COLORKEY

Initialize color-key information from opened image. This flag is mainly used for proper conversion conversion to compressed texture formats, where color-key information is specially encoded.

IMGOPEN_HINTDYNAMIC

A hint to the system, that image will be often accessed, thus should be placed in memory optimized for fast and frequent update. Usually used with IMGOPEN_TEXTURE flag.

IMGOPEN_HINTSTATIC

When specifying this flag, the image data are static - stored in memory optimized for fast use of image (texture), but writes to it may be slower. Usually used with IMGOPEN_TEXTURE flag.

IMGOPEN_MEM_PTR

The bitmap data are not read from file, but rather are already stored in memory, and caller provides a pointer to this memory. Currently this flag works only for DIB sections stored in memory (BMP format).

IMGOPEN_RENDERTARGET

Requests creation of image, will be usable as render target.

IMGOPEN_INIT_CKEY

Init colorkey alpha from color #0, create RLE transparency information. When using this flag, the library builds color-key information kept with the image, which may be accessed by application.
