S_pixelformat

This structure is used for representation of surface pixel format. The pixel format may describe format of images, textures, buffers in swapping chains, etc.
The structure is defined as follows:

struct S_pixelformat{

   union{

      dword bytes_per_pixel; 

      dword four_cc;          

   };

   union{

      struct{

         dword r_mask;

         dword g_mask;

         dword b_mask;

         dword a_mask;

      };

      struct{

         dword u_mask;

         dword v_mask;

         dword l_mask;

 

      };

   };
   dword flags;

};

 

typedef S_pixelformat *PS_pixelformat;

Members:

bytes_per_pixel

Bytes used for making up single pixel. This value is valid for non-Four-cc pixel formats.

four_cc

Four-CC code, used to represent special non-RGB surfaces.

r_mask, g_mask, b_mask, a_mask

Bit-masks representing bits in pixel assigned to Red, Green, Blue and Alpha channels. These values are valid only for RGB or ARGB formats.

u_mask, v_mask, l_mask

Bit-masks representing  bits in pixel assigned to U, V and Luminance channels. These values are valid only for Bump-map pixel formats.

flags

Flags describing the pixel forma.
This value may be combination of following flags:

PIXELFORMAT_PALETTE

Palletized pixel format. This format is one bite per pixels, and each pixel is encoded as index into color palette consisting of 256 entries.

            PIXELFORMAT_ALPHA

Pixel format contains alpha channel.

            PIXELFORMAT_COMPRESS

Compressed pixel format. This format is represented by four_cc code.

            PIXELFORMAT_BUMPMAP

Bump-mapped pixel format.
