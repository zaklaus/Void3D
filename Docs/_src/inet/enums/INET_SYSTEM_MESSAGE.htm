INET_SYSTEM_MESSAGE

The enumerated type INET_SYSTEM_MESSAGE specifies system messages posted to user callback function. The system message is posted similar way as data from other network player, the difference is that they are generated locally by I_net interface, and they are not sent across network. The data are fetched from the C_net_in_msg interface.
To detect if message is standard or system one, user should look at ID of sender in the INET_MESSAGECALLBACK function; if the sender ID is zero, the message is system message, otherwise the message is received from specified player.

In this description, some messages are followed by additional data. These data are stored in the input message queue, and immediately follow the message.

Following members are declared in the enumeration:

INET_SYSMSG_PLAYER_CREATED

New player was created.
Following is a dword parameter specifying the ID of the newly created player. Using this ID, you may determine the name and other parameters of the new player using standard methods of I_net interface.

INET_SYSMSG_PLAYER_DESTROYED

Existing player was destroyed.
Following is a dword parameter specifying the ID of the player. Following this is a null-terminated string specifying the name of destroyed player. This is due the reason that you cannot determine the name of already destroyed player.

INET_SYSMSG_HOST_MIGRATED

Host migrated. The migration is allowed only if it is allowed – see INET_INIT for details.
Following is a dword value specifying the ID of the new host.

INET_SYSMSG_SESSION_OVER

Session was disconnected and closed.
