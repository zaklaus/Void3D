I3D_interpolator

I3D_interpolator is an interface, which is responsible for interpolation between animation keys stored in I3D_animation_base and apply result to frame. It keeps references to both I3D_animation_base and I3D_frame during its lifetime. The I3D_animation_base is used as a read-only resource, so it is possible to use the same I3D_animation_base with multiple interpolators. All variables needed to interpolate are property of concrete I3D_interpolator.

Each interpolator can interpolate one or more animations (I3D_animation_base). If multiple animations are interpolated, they may be blended together before their result is applied to frame. Each animation is processed in separate stage, each stage having its own animation and a unique blend value.

Animations are processed from stage with lowest index upwards. Each two stages with non-zero animation are blended by blend value of higher stage in following ratio:
resulting_animation = animation[n-1] * (1.0 - blend_ratio[n]) + animation[n] * blend_ratio[n].
This implies that blend ratio of very first non-zero stage is ignored. The result of two stages is then blended with next stage using the same formula, until all stages are processed. The result is applied to I3D_frame. The result may be position, rotation, scale or other frame-specific value, such as color for lights, or volume for sounds, etc.

Limitations: only position, rotation and scale is processed in multiple stages and blended together. Other tracks are processed only on animation set to stage 0.

Remarks:

When Tick is called, the interface processes all non-empty tracks in the animation. The computed values (which are values dependent on type of animation track) are applied to frame by methods of I3D_frame, such as for example I3D_frame::SetPos to set new position.
