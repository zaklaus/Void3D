I3D_sound

I3D_sound represents sound object. This interface inherits from I3D_frame interface.
The sound is automatically updated when scene is rendered. It uses either sound hardware if this supports 3D hardware buffers, or it performs emulation to set balance and volume of sound. The virtual 'listener' is in camera position. Sounds that are beyond their audible distance are automatically paused so that they don't consume CPU processing time and hardware resources. The system takes care of re-using hardware 3D buffers for sounds which are currently audible.

Sounds and sectors:
A sound is audible only when listener (camera) is in one of sectors on which the sound is registered. See I3D_Sector::AddSound and I3D_sector::RemoveSound for details. One exception to this rule is when sound is not registered in any sector, in which case it is audible in all sectors.

Remarks:

In general, only mono sounds should be used for I3D_sound that are 3D positioned in space. Stereo sound sources are suitable only for ambient sounds.

The visibility flag set for this interface using the I3D_frame::SetOn method, set the status of sound to be either playing or stopped. If non-looped sound has finished playing, it may be reset by calling I3D_frame::SetOn with true value to reset sound playback.
