I3D_sector

This interface inherits from I3D_frame interface.
I3D_sector is advanced frame type used to partition 3D space into logical sectors. Its main purpose is to speed up rendering, as it helps cull out groups of attached child frames if the sector is not visible. In combination with portals it performs even more complex hidden object removal of object contained in sector.
The second advantage of sector is possibility to define list of lights and sounds that are applied to particular sector, which helps to make better realism to 3D scene by allowing to set different lights and/or sounds to different sectors. This includes all light types, including ambient and fog lights. This also affects performance of rendering by decreasing number of lights applied to visuals during rendering.

Remarks:

Sector definition.
Sector is defined by a space, which it occupies. The space must be convex - this means it can't contain corners with non-convex angles. The convex hull is computed when sector is created, from a set of given frames (walls). When convex hulls of two or more sectors overlap, the behavior in the overlapping is undefined - this includes detecting sector to which belongs a 3D point, sounds and lights.

Primary sector, backdrop sector and other sectors.
There are two implicitly created primary sector in the I3D_scene object.
First is primary sector, which is the root of frame hierarchy tree. It has infinite bounding volume, and represents all the sector space not occupied by other sectors. There is only one primary sector in each scene. Any other frames which are to be rendered are children (direct or indirect) of the primary sector. Primary sector can't be created nor destroyed by user.
The second is backdrop sector, which is the root oh frame hierarchy to be rendered as backdrop. It may contain lights, which are applied on children of the sector. The backdrop sector is rendered using the same field-of-view as current scene's camera, but may have different ranges, which are set by related methods of I3D_scene.

Portals.
Portals are holes in sector skeleton. These are the holes by which camera may see inside of sector or out of sector. Each sector has its own portals - the ones which it has created during build, and connected portals - the ones by which it is connected with other sectors. Portals may be implicitly enabled or disabled by user. See I3D_portal for details.
