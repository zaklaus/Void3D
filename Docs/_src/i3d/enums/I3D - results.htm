I3D return codes

Following are return codes from I3D library functions. The type of return code is I3D_result.
You may use macros I3D_SUCCESS and I3D_FAIL to determine if return code is a success or fail return code.

I3D_OK

The operation completed successfully.

I3D_DONE

The operation completed successfully, and is definitely finished. This is the case of Tick-ing animation, for example.

I3DERR_NOFILE

Unable to open specified file.

I3DERR_OUTOFMEM

Operation was unsuccessful because of memory allocation failure.

I3DERR_TEXTURESNOTSUPPORTED

The selected 3D driver doesn't support textures, or no appropriate texture format was found.

I3DERR_GENERIC

Internal error.

I3DERR_OBJECTNOTFOUND

Object not found.

I3DERR_INVALIDPARAMS

Some of input parameters are invalid.

I3DERR_CANTSETSECTOR

The sector cannot be set.

I3DERR_NOTINITIALIZED

The object is not initialized.

I3DERR_FILECORRUPTED

File open fail - the file being loaded is corrupted.

I3DERR_NOFILE1

Unable to open specified file provided as second parameter - valid for opacity maps.

I3DERR_CYCLICLINK

The frame which user attempts to link to is already child of the frame, so linking would cause cyclic hierarchy dependency.

I3DERR_CANCELED

The operation was canceled by user. This is code returned if user returns I3DENUMRET_CANCEL in enumeration callback procedure.

I3DERR_UNSUPPORTED

Requested operation is not supported.
