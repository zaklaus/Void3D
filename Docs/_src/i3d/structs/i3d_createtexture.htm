I3D_CREATETEXTURE

The structure I3D_CREATETEXTURE is used when specifying parameters for creating texture by I3D_driver::CreateTexture function.
The structure is declared as follows:

struct I3D_CREATETEXTURE{
   dword flags;
   union{
      const char *file_name;
      C_cache *ck_diffuse;
   };
   union{
      const char *opt_name;
      C_cache *ck_opacity;
      dword proc_data;
   };
   dword size_x, size_y;
   S_pixelformat *pf;
};

Members:

flags

Creation flags. Following flags are defined:

TEXTMAP_DIFFUSE

Creation of texture with diffuse bitmap requested, the name is given through txt_name or ck_diffuse members.

TEXTMAP_OPACITY

Creation of texture with opacity map requested, the name is given through opt_name or ck_opacity members. The opacity map is encoded into the alpha channel of the texture.

TEXTMAP_EMBMMAP

Creation of environment-mapped bump-map texture requested, the name is given through file_name member.

TEXTMAP_NORMALMAP

Creation of normal-map texture requested, the name is given through file_name member.

TEXTMAP_CUBEMAP

Request for creation of cubic texture. The name represents the base of name. Six names of texture sides are automatically generated during loading, by adding “_l”, “_r”, “_t”, “_b”, “_f” and “_k” suffixes, for left, right, top, bottom, front and back side bitmaps of the cube map.

TEXTMAP_TRANSP

Request for initialization of color-key information from the source bitmap. The colorkey information is encoded into texture (for some pixel formats), and texture is marked as colorkeyed.

TEXTMAP_MIPMAP

Request for automatic generation of mipmap levels during texture loading.

TEXTMAP_NOMIPMAP

Disable automatic generation of mipmap levels on hardware which automatically creates mipmaps.

TEXTMAP_COMPRESS

Choose compressed texture format, if available.

TEXTMAP_USEPIXELFORMAT

Use pixelformat specified by pf member. For details, see S_pixelformat.

TEXTMAP_HINTDYNAMIC

Hint to the system, that the texture will be often locked and modified.

TEXTMAP_TRUECOLOR

Request true-color pixel format for texture, if available by hardware, rather than high-color or palettized. This flag disables texture compression.

TEXTMAP_USE_CACHE

Diffuse and opacity maps are loaded using provided C_cache interface, rather than loaded from disk.

TEXTMAP_NO_SYSMEM_COPY

Do not create system-memory copy of texture. This flag is allowed only for empty textures (no source data provided). Also mipmaping cannot be used with this flag. This flag is intended for procedural textures.

TEXTMAP_PROCEDURAL

The texture will be created using procedural, not by loading data from file. The procedural must specify procedural texture.

file_name

Pointer to filename, from which texture will be created. Depending on flags, this may be either name of bitmap, or name of procedural.

ck_diffuse, ck_opacity

Pointer to C_cache class, specifying source of texture data. It is assumed, that current read position of the cache class is at beginning of image data.

proc_data

Additional data used for creation of procedural. This member must be set to one of I3D_MATERIAL_TEXTURE_INDEX enumerated type.

size_x, size_y

Size of textures. This information is used if neither of TEXTMAP_DIFFUSE, TEXTMAP_OPACITY or TEXTMAP_BUMPMAP flags was used, in order to determine the requested size of created texture. These members are used when creating empty textures.

pf

Pixel format in which texture will be created. This member is valid only if TEXTMAP_USEPIXELFORMAT flags is specified. If TEXTMAP_USEPIXELFORMAT is omitted, the system chooses best pixel format for the texture.
