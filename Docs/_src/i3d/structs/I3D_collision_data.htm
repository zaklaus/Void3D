I3D_collision_data

I3D_collision_data structure is used for specifying input parameters for collision testing, and for determining information after collision testing has completed.
This structure is used for any kind of collision testing supported by the engine.

The structure is declared as follows:

struct I3D_collision_data{
   S_vector from;
   S_vector dir;
   dword flags;
   PI3D_collision_response callback, texel_callback;
   void *cresp_context;
   CPI3D_frame frm_ignore;
   float radius;
   PI3D_frame frm_root;
   const PI3D_volume *vol_group;
   dword num_vols;
   dword collide_bits;
};

Members:

from

Start position, from which collision will be tested.

dir

Direction, at which collision test will be performed. This value is used only for tests which test movement (i.e. it is ignored for tests detecting static collisions).
The magnitude of this vector determinates length, until which testing will occur. This may be overridden by specifying I3DCOL_RAY flag.

flags

Flags specifying how the test will be performed. Lower 16 bits are reserved for mode, while higher 16 bits are reserved for additional flags, which may be combined with the mode.

Collision test mode (the modes cannot be combined in one call):

I3DCOL_LINE

Line tested against BSP + dynamic volumes.

I3DCOL_MOVING_SPHERE

Moving sphere tested against BSP + dynamic volumes.

I3DCOL_MOVING_GROUP

Test all owner's sphere volumes as bounded group. In this mode, each face if tested for entire group only ones. Thus if you reject collision for one vol_source, do not expect response it for other one in the same test.

I3DCOL_STATIC_SPHERE

Static sphere against BSP tree.

I3DCOL_VOLUMES

Dynamic test on all scene volumes regardless on static flag. (no BSP usage).

I3DCOL_EDIT_FRAMES

Test line against currently visualized edit frames (lights, sounds, ...).

I3DCOL_NOMESH_VISUALS

Line test against bounding boxes of mesh-less visuals (particles, flares, etc).

I3DCOL_STATIC_BOX

Test if box collide with other dynamic volumes. Currently only test against dynamic spheres supported.(no boxes, no BSP, no static spheres).

I3DCOL_EXACT_GEOMETRY

Line test against all geometry, ignoring BSP tree. Slow test, suitable for editing purposes.

 

Following additional flags may be combined with the mode flag:

I3DCOL_RAY

Test ray instead of line segment (ignore length of input direction vector). Valid for moving tests only.

I3DCOL_COLORKEY

When checking geometry, inspect texel on collided position, ignore transparent (color-keyed) textels.

I3DCOL_INVISIBLE

Test also invisible frames.

I3DCOL_FORCE2SIDE

Always consider faces to be 2-sided, regardless of material (also volumes).

I3DCOL_SOLVE_SLIDE

Compute sphere slide along collision planes (moving sphere tests only).

I3DCOL_JOINT_VOLUMES

Test also joint volumes on single-mesh. Valid only for moving test modes.

callback

Called after every collision with face. See I3D_collision_response and I3D_cresp_data for more details.

texel_callback

Called after every line-texel test. Internally used for ligthmapping. Line test only when I3DCOL_TEXTURE is specified.

cresp_context

User collision context. Passed from collision function to callback functions (I3D_collision_response).

frm_ignore

This frame and all it's children are skipped in tests. Currently used only in Line tests.

radius

Radius of tested sphere. Used only in modes I3DCOL_MOVING_SPHERE or I3DCOL_STATIC_SPHERE.

frm_root

Frame, which will be enumerated for all sphere volumes and those will be tested as group. Used only with I3DCOL_MOVING_GROUP.

vol_group

Alternative to frm_root. Instead specify frame with which has volumes, it allow direct pass array of volumes, which will be tested as group. Used only with I3DCOL_MOVING_GROUP.

num_vols

Number of volumes in the array pointed to by vol_group.

collide_bits

Category collision bits. These bits are checked agains category bits in tested collision volume during tests (by logical AND), and if at leas bit matches, the collision with particular volume is accepted. Otherwise the tested volume is ignored.

Methods:

Constructors:

I3D_collision_data();

Default constructor. This method resets all members of the class to zero.

 

I3D_collision_data(const S_vector &from, const S_vector &dir, dword flags, PI3D_frame frm_ignore = NULL);

Constructor setting up internal variables of the class.

Retrieving values:

float GetHitDistance() const;

Return distance of closest collision point. Collision distance is normalized (you need normalize dir to compute correct hit position).This method returns valid result only if collision was detected. When I3DCOL_SOLVE_SLIDE is specified, result is undefined.

 

S_vector ComputeHitPos() const;

Compute and return collision point. It uses formula from + dir*GetHitDistance() / dir.Magnitude(), just make computation instead user. This method returns valid result only if collision was detected.

 

PI3D_frame GetHitFrm() const;

This method returns closest frame, which was hit.

 

const S_vector &GetHitNormal() const;

Return normal of collision. This method returns valid result only if collision was detected.

 

const S_vector &GetDestination() const;

Return solved position after collision response (user or default sliding). Calling this method is applicable only after I3DCOL_SPHERE or I3DCOL_SPHERE_GROUP collision tests.

 

int GetFaceIndex() const;

Index of collision face into frame’s mesh or (–1) if collision frame has no mesh. It can be used for obtain more information like face, facegroup or material.

 

const S_vector &GetNormalizedDir() const;

Get unit-length version of direction vector passed in dir member. This method returns valid vector only after TestCollision was called – this means either when the function returned, or in callback function.
