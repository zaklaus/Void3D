I3D_scene::CreateBspTree

 
Compute BSP tree from volumes (I3D_volume) and visuals (I3D_visual), which are marked for static collision testing using I3D_frame::SetFlags, having set I3D_FRMF_STATIC_COLLISION flag.
 

I3D_RESULT CreateBspTree(

            I3D_BSP_BUILD_CB_PROC *cbP = NULL,

            void *context = NULL

);

Parameters:

cbP

Callback function which will be called to display creation progress. The callback function prototype is declared as:

bool I3DAPI I3D_BSP_BUILD_CB_PROC(I3D_BSP_MESSAGE msg, dword prm1, dword prm2, void *context);

Parameters to callback functions:

msg

Message sent to function. This may be one of following:
BM_GETTING_POLYGONS
   Polygons are collected (prm1 represents current polygon count)
BM_BUILDING_NODES
   Nodes are built (prm1 represents proposed number of polygons left unprocessed)
BM_BUILDING_EDGES
   Edges are being built (prm1 represents percentage of the operation)
BM_LOG
   Log message (prm1 is pointer to message string, prm2 is priority – 0=error, 1=warning)

prm1, prm2

Parameters being used in dependency of msg parameter.

Return value:

The returned value is currently ignored, but it is recommended to pass true as return value.

context

Context passed to the callback function.

Return Value:

The returned value is I3D_OK if the call was successful.

See Also:

DestroyBspTree, I3D_frame::SetFlags

I3D_scene::LoadBsp

This function is used for loading BSP collision tree from a file.

bool LoadBsp(
            C_cache *file,
            I3D_LOAD_CB_PROC *cb_proc =  NULL,
            void *cb_context = NULL,
            dword check_flags = 0
);

Parameters:

file

Pointer to C_cache interface, from where BSP data will be loaded.

cb_proc

Callback function used for displaying progress during loading. See I3D_LOAD_CB_PROC for details.

cb_context

Context passed to callback function.

check_flags

FBSPLOAD_CHECKSUM

Compare checksum on each frame for detect possible changes between scene as was during build and now.
If differences finded, tree is destroyed. For this purpose should not be used in final release of game, where bsp MUST be valid.

FBSPLOAD_CONSISTENCE

Make additional consistency checking, report all frames which are static and not yet included in bsp tree.
Should not be used in final game, it slow down loading.

Return Value:

The returned value is I3D_OK if the call was successful.

See Also:

SaveBsp, IsBspBuild, I3D_LOAD_CB_PROC, C_cache

I3D_scene::DrawSprite

 
Draw 2D sprite image in 3D space.
 

I3D_RESULT DrawSprite(

            const S_vector &pos,

            PI3D_material material,

            dword color = 0xffffffff,

            float size = 1.0f,

            const I3D_text_coor *in_uv = NULL

) const;

Parameters:

pos

Position where sprite will be drawn, in world coordinates.

material

Material used to render the sprite.

color

Color used for rendering the sprite, in ARGB format.

size

Size of side of sprite, which is transformed by camera's projection matrix.

in_uv

Pointer to pair of UV coordinates of rectangle within the material’s texture. The rectangle specifies left-top and right-bottom corners in UV space. If this parameter is NULL, the rectangle is mapped to [0, 0] and [1, 1] uv coordinates.

I3D_scene::GetLightness

 

Get lightness on specified position of scene. First, a sector is determined (using GetSector function), where specified point lies, and all matching lights set to that sector are used in computation.

 

S_vector  GetLightness)(

            const S_vector &pos,

            const S_vector *norm = NULL,

            dword light_mode_flags = I3DLIGHTMODE_VERTEX

);

Parameters:

pos

Point to get lightness for, in world coordinates.

norm

Normal of tested point.

light_mode_flags

Flags specifying which lights to consider for the operation. See I3D_light::SetMode for details on possible flags. All light matching at least one provided flag will be used in computation.

Return Value:

The return value is a vector specifying RGB values of lightness. All values may be in range from 0.0f to 2.0f.

See Also:

GetSector, I3D_light::SetMode

I3D_scene::UnmapScreenPoint

 

Inverse-map 2D screen point to a ray defined by a start position (which is position of camera if camera is perspective) and a direction vector. The returned direction vector is normalized.

 

I3D_RESULT UnmapScreenPoint(

            int x,

            int y,

            S_vector &pos,

            S_vector &dir

);

Parameters:

x , y

Input 2D screen point.

pos

Resulting line position, at which look ray starts. This is actually position of current camera if perspective projection is used.

dir

Resulting unit-length direction of ray, going through specified screen point.

Return Value:

The returned value is I3D_OK if the call succeeded.

See also:

TransformPoints
