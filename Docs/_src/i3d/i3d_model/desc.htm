I3D_model

I3D_model is a frame, which contains list of other frames. This interface inherits from I3D_frame interface.
This interface also contains all methods of I3D_container subclass.

Model naming conventions.

The Insanity3D identifies frames by their names. It is possible to have two frames having the same name, anyhow it may cause problems for user during frame enumeration, and other tasks performed by user (such as saving references to frames to disk). The problem with name duplication may arise in case of frames owned by model when two models open the same file. Therefore names of frames owned by model are modified each time I3D_frame::SetName is called in the model frame. In this call, model’s name is set normal way, and all owned frames are renamed to have model’s name as base, followed by dot (.) and followed by their original name. For example, if model is renamed to "Shooter" and the model contains a frame originally named "gun", this frame will become a name "Shooter.gun".

This system provides that all names of frames should be unique unless user explicitly sets identical name for two different frames.

To obtain the original name of frame, this means the one set by I3D_frame::SetName, the user may call I3D_frame::GetOrigName. Note that this name will be identical to those returned by I3D_frame::GetName for all frames which are not owned by a model.

The visibility flag set for the I3D_model interface is used for enabling or disabling rendering of all visual frames linked to this frame.
