I3D_frame

I3D_frame interface is a base class for other interfaces. It represents an object in the scene hierarchy tree, and contains all basic functionality for such an object. See Remarks section for more information.

Remarks:

Each node in the scene tree inherits from I3D_frame base object. I3D_frame keeps information about node’s position, rotation, scale, it's type and name. Additionally it keeps a visibility flag, and 32-bit value free to use by user.
Such a scene node is referred to as frame in entire Insanity3D library.

Frames matrices.

Each frame internally holds its local matrix, and a resulting matrix that is dependent on hierarchy tree. These matrices are read-only for user; their computation is handled internally, and is affected by position, rotation and scale of frame and by matrix of parent object.
Additionally, I3D_frame holds inverted matrix. All matrices are computed as necessary, and are updated when appropriate.

Frame hierarchy.

Frames in 3D scene are stored in hierarchy tree. Each frame may have one or no parent and zero or more children. At the top of hierarchy tree is a root frame. This root frame is a sector called Primary sector. In order for frames to be rendered they must be children of root frame, either directly, or children of its ancestors. Position, rotation and scale are inherited from parent frame by its child frames.

Visibility flag.

Each frame has a visibility flag (boolean type), which meaning is depends on the type of particular frame. For FRAME_VISUAL its meaning is for determining frame’s visibility. For FRAME_SOUND it means if sound is audible. Concrete use of flag is described along with particular frame type for which it has meaning.

Frame types

Specialized classes inherited from I3D_frame are identified by frame type. See I3D_FRAME_TYPE for details.

Casting frame types

To cast pointer to I3D_frame to a pointer to derived frame, user may use one of cast macros, which perform safe cast. Casting is useful in such places where user knows exactly the type a a frame (I3D_frame::GetType), and wants cast the pointer to I3D_frame to a pointer to specialized frame.

Following cast macros are defined:

#define I3DCAST_LIGHT(f)      (static_cast<PI3D_light   >((PI3D_frame)f))
#define I3DCAST_SOUND(f)      (static_cast<PI3D_sound   >((PI3D_frame)f))
#define I3DCAST_CAMERA(f)     (static_cast<PI3D_camera  >((PI3D_frame)f))
#define I3DCAST_OBJECT(f)     (static_cast<PI3D_object  >((PI3D_frame)f))
#define I3DCAST_LIT_OBJECT(f) (static_cast<PI3D_lit_object>((PI3D_frame)f))
#define I3DCAST_DUMMY(f)      (static_cast<PI3D_dummy   >((PI3D_frame)f))
#define I3DCAST_JOINT(f)      (static_cast<PI3D_joint   >((PI3D_frame)f))
#define I3DCAST_USER(f)       (static_cast<PI3D_user    >((PI3D_frame)f))
#define I3DCAST_VOLUME(f)     (static_cast<PI3D_volume  >((PI3D_frame)f))
#define I3DCAST_SECTOR(f)     (static_cast<PI3D_sector  >((PI3D_frame)f))
#define I3DCAST_MODEL(f)      (static_cast<PI3D_model    >((PI3D_frame)f))
#define I3DCAST_VISUAL(f)     (static_cast<PI3D_visual  >((PI3D_frame)f))
#define I3DCAST_OCCLUDER(f)   (static_cast<PI3D_occluder>((PI3D_frame)f))

It is recommended to use these macros to cast PI3D_frame to pointers to derived classes.

Frame enumeration.

Each frame may have other frames attached to itself in hierarchy tree. The user may enumerate attached objects in the tree using EnumFrames method. During enumeration, provided call-back functions is called for each attached frame matching specified criteria. The method FindChildFrame is used to find frame in frame's hierarchy tree based on specified criteria. These criteria are the same as for EnumFrames method.

Remarks:

The name comparison on frames owned by model is performed on their original names. See GetOrigName for details.
