Lightmapping

 

 

Introduction

 

Light-mapping is a technique used for improving visual appearance of scenes. If is used for static geometry, where light information may be pre-computed at design time. The lighting is stored using detailed lightmap textures (the resolution of these textures is adjustable by designers). This provides much better detail than vertex-based lighting.

The engine supports light mapping through specialized visual called lit object, which is similar to standard rendered visual object, but it manages information about lightmapping, and is capable to display geometry using the maps.

 

Insanity provides computation of lightmap individually on any lightmapped object, or groups of objects, using the Insanity editor. The engine uses lights placed in scene, and computes shadows from geometry objects obstructing light rays falling onto the lightmapped visual. With lightmapping, there’s no limit on number of lights shining on single lightmapped visual.

 

 

 

Technical implementation

 

The lightmapping is applied on selected visuals in the scene. The visuals which are desired to be lightmapped objects need to be cast to the lit object visual. The lit object may be in two modes – either in initial state, where lightmapping is not computed (in this case object is rendered using special shade), or in normal state with computed lightmaps.

Internally, when lightmap is computed on such object, several computations are performed. The geometry of object need to be analyzed, and the engine prepares list of rectangles, which best map onto existing geometry. Each rectangle may cover one or more triangles in the visual’s mesh, depending on their angle, smooth-groups, normals, etc. This rectangle information is stored in the visual. Each rectangle has assigned a grid of lightmap pixels.

Another step is computation of lighting for all rectangles, for each pixel. During this step, all lights applying on a rectangle is taken, and collision testing is performed for each ray. The information is stored in the lightmapped rectangles.

The lightmap computation uses antialiasing, which significantly improves areas where shadows and light meet.

 

When lightmapped visual is about to be rendered, its lightmap rectangles are uploaded to lightmap texture cache (which are several large textures used for caching rectangles from multiple visuals), and its vertices are mapped by correct uv coordinates in the lightmap cache texture. This step is done once before rendering. Then visual is normally rendered, using base texture and the lightmap texture.

 

 

 

Lightmap construction

 

Lightmaps are constructed using the Insanity editor. First it is necessary to select objects which are to be lightmapped visuals. Then they must be changed from standard visual objects into lightmapped visuals using the Properties window. In the Properties window, designers may choose lightmap resolution, which is measured as number of lightmap pixels per one scene world unit.

Finally, lightmaps are computed using the Light-map menu. The menu works with selection, or all lightmapped visuals in scene, so it is possible to adjust one or more lights and re-compute lightmapping on selection only.

 

 

 

Saving/loading lightmaps

 

Lightmaps are saved into mission binary file, when mission is saved, and they are applied back during loading. When the geometry of lightmapped visual changes, the computed lightmaps will fail to load, and they will need to be recomputed.
