Plugin:

Undo

Description:

Undo/redo functionality. Very simple, but essential plugin for saving undo and redo information, and recalling it. It works without knowing any information about other plugins.

Editor Interface:

The undo plugin is usable directly only by two shortcuts which it exposes – Undo and Redo. These two simple operations should traverse through scene modification actions performed by user, and allow user to return to any previous point in editing.

Modifying scene (writing into undo buffer) clears the redo buffer.

Programming API:

When a plugin modifies something in the scene, it opens a record in the Undo buffer, utilizing C_chunk cache interface, and writes any information which is to be saved. Careful should be taken when saving pointers (which may not be valid when action is undone).

Each undo entry may be associated with one I3D_frame interface, which is passed to Open method, and Undo plugin keeps reference of it.

When user selects Undo/Redo, the registering plugin’s Undo method is passed, with the data in the chunk which it created, and pointer to I3D_frame, which is associated with the action.

After each scene Tick, a special mark is written to the internal undo buffer, which marks an end of frame. When undo is performed, multiple actions are undone/redone, until next such mark. This way, other plugins may create multiple entries, which will be undone at one user’s click.

 

 

Plugin:

Selection

Description:

Keeping current selection of objects in the scene, displaying selection dialog, sending notifications when selection changes.
Additionally, this plugin displays flashed frames – objects which bounding box flashes for various reasons – to mark error, to notify user that something happened with the object, etc.

Editor Interface:

For user, this plugin provides logical selection of one or more I3D_frame objects in the scene. The selection is marked by rendering bounding boxes around selected objects, and optionally link to parent’s frame. This plugin also provides the selection dialog, where objects may be selected using various categories or by name.

Programming API:

Other plugins may retrieve vector containing current selection. They may also install notification callbacks, which are called when selection changes.

 

 

Plugin:

Modify

Description:

Keeping modification info associated with each frame. It also stores initial values of modified parameters, so that the parameters may be reset to original ‘after load’ state.
This plugin is responsible for saving modification information of mission.

Editor Interface:

-

Programming API:

This plugin is used mainly by other plugins, in order to store modification information in one place. There’re several flags named E_MODIFY_FLG_XXX (where XXX is replaced by actual flag name), which define type of modification made on frame, for example position. Each modification on a frame may be added or reset. When adding modify flag, the plugin saves original “non-modified” value of the modification, so that the value may be reset later to original state.

This plugin also saves current modification into mission binary file. The loading is done by game itself, which must add methods of this plugin during loading in order to allow editing of scene.

 

 

Plugin:

Create

Description:

Creation and deletion of frames. This plugin also provides browse dialogs for selecting sounds and models.

Editor Interface:

Through Create menu, or keyboard shortcuts, user may create various types of frames. For particular types, user is asked for additional parameters during creation.

Plugin also offers deletion and duplication of frames.

Programming API:

Possibility to use plugin’s browser dialogs for selecting model or sound, and deletion of particular frame (letting the plugin saving undo information).

 

 

Plugin:

MouseEdit

Description:

Editing and flying around the scene.

Editor Interface:

This plugin is the main interface between user and scene, translating keyboard and mouse actions into commands manipulating the scene (moving, rotating, scaling, flying, etc). Interface provides mainly methods for comfortable navigation through scene, and modification of scene objects using the mouse.

Programming API:

Other plugins may utilize this plugin for interaction-based tasks. They may hook the mouse, change cursor, install callback for user’s click, etc.

 

 

Plugin:

Properties

Description:

Plugin managing the Properties dialog window.

It also allows other plugins to install their tabs in the Properties window.

Editor Interface:

This plugin provides additional Properties dialog window, typically always displayed in editor, which provides interactive information about current actions. This plugin provides information about current selection (tabs for frame, and specialized frames such as sounds, lights, etc). Using this plugin, user may modify scene by typing numerical values into the dialog.

Plugin itself provides sheets for displaying and modifying basic parameters of all frame types.

Programming API:

Various plugins also use the property window for installing its own dialog windows. The properties window may also be made the activate window.

Mission

Manager of mission loader and browser.

Mission manager. Allows browsing through scenes, using source control, starting/ending game mode, etc.

Programming API:

The editor provides just base class, not implementation. Must be implemented by game application.

 

 

Plugin:

Log

Description:

Managing the Log output.

Editor Interface:

The plugin works using its sheet in the Properties window. It provides output only, and allows saving the contents of the log window into a file.

Programming API:

Many plugins use the possibility of this plugin to output many kinds of information, which should be displayed to user. The Log window supports colored texts, as well as rich-text.

Description:

CamPath

Description:

Camera path animator. Plugin for animating camera path movement on bezier curves. Used for creation of camera movement directly in 3D scenes, with settings of times, delays, notify keys.

Editor Interface:

Animation tools for creating camera path movement, using specialized dialog, and visualization of camera path and keys. Plugin allows playback of edited path.

Programming API:

A class loading the camera path from binary file is provided.
